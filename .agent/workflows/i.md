Цей skill призначений для імплементації функціональності згідно з планом та принципами TDD.

### Інструкції:
1. Прочитати [SPECIFICATION.md](file:///c:/Users/adm/workspace/whitelist_editor/SPECIFICATION.md), щоб пригадати загальні вимоги.
2. Ознайомитися з затвердженим планом роботи у файлі `plan/PLAN.md`.
3. Використовувати підхід **TDD (Red-Green-Refactor)**:
    - **Red**: Написати тест для нової функціональності, який не проходить (або не компілюється). Хороший тест перевіряє призначення коду (intent), а не реалізацію.
    - **Green**: Написати мінімально необхідний код, щоб тест пройшов.
    - **Refactor**: Провести рефакторинг коду, зберігаючи працездатність тестів.
4. Реалізувати необхідну функціональність повністю згідно з планом.
5. Ретельно перевірити роботу програми використовуючи `cargo test` та `cargo clippy` та виправити всі помилки та попередження.
