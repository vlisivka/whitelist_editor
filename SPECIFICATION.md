# Специфікація проєкту Whitelist Editor

## Опис проєкту
Цей проєкт — це десктопний додаток для керування статичними DHCP-призначеннями (leases) на роутерах MikroTik. Він призначений для редагування статичних адрес, які роздаються абонентам через сервер DHCP, запущений на роутерах MikroTik.

## Основні вимоги
- **Цільова аудиторія:** Проєкт створений спеціально для україномовних споживачів. Інтерфейс та документація мають бути українською мовою.
- **Методологія розробки:** Використовується **TDD (Test-Driven Development)**. Будь-яка нова функціональність повинна починатися з написання тестів.
- **Технологічний стек:**
    - Мова програмування: **Rust**.
    - Графічний інтерфейс: **egui**.
    - Комунікація з роутером: **SSH** (бібліотека `ssh-rs`).
- **Ключові функції:**
    - Виконання команди `/ip dhcp-server export` на роутері та парсинг результатів.
    - Відображення списку серверів, мереж та лізів у зручному інтерфейсі.
    - Валідація IP-адрес: перевірка на коректність формату, унікальність та приналежність до мережі відповідного DHCP-сервера.
    - Автоматичне знаходження першої вільної IP-адреси в мережі при додаванні нового абонента.
    - Синхронізація змін з роутером.

## Структура проєкту
Проєкт організований за модульним принципом:

- [main.rs](file:///c:/Users/adm/workspace/whitelist_editor/src/main.rs): Ініціалізація додатку та запуск циклу `eframe`.
- [app.rs](file:///c:/Users/adm/workspace/whitelist_editor/src/app.rs): Головний модуль GUI. Описує стан програми, візуальні компоненти (таблиці, форми редагування) та обробку подій.
- [mikrotik_data.rs](file:///c:/Users/adm/workspace/whitelist_editor/src/mikrotik_data.rs): Моделі даних (`Lease`, `DhcpServer`, `DhcpNetwork`) та бізнес-логіка. Тут реалізовано парсинг конфігурації MikroTik та алгоритми валідації. Містить набір unit-тестів.
- [ssh_client.rs](file:///c:/Users/adm/workspace/whitelist_editor/src/ssh_client.rs): Модуль для роботи з SSH-з'єднанням. Відповідає за авторизацію та виконання команд.
