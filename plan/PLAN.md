# План додавання підтримки нестандартного SSH порту

Цей план передбачає можливість вказання SSH порту безпосередньо в полі адреси хоста (наприклад, `192.168.88.1:2222`).

## 1. Зміни в `ssh_client.rs`
- [ ] Оновити метод `SSHClient::connect`, щоб він перевіряв наявність порту в рядку `host`.
- [ ] Використовувати `rsplit_once(':')` для безпечного відділення порту.
- [ ] Якщо порт вказано (через двокрапку), використовувати його.
- [ ] Якщо порт не вказано, за замовчуванням використовувати 22.

## 2. Перевірка
- [ ] Протестувати підключення з дефолтним портом (просто IP).
- [ ] Протестувати підключення з вказаним портом (IP:port).

## Питання до уточнення:
- Чи потрібно підтримувати IPv6 адреси в полі "Мікротик"? (Для IPv6 з портом зазвичай використовується формат `[addr]:port`). Якщо ні, то простої перевірки на `:` достатньо.
- Відповідь: Так, потрібно підтримувати формат IPv6.
