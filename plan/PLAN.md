# План впровадження екранування рядків MikroTik

Цей план передбачає реалізацію коректної обробки спецсимволів та Unicode (UTF-8) при читанні даних з MikroTik та відправці команд назад.

## 1. Дослідження та розробка логіки екранування (`mikrotik_data.rs`)

### 1.1. Покращення парсингу значень
- [ ] Оновити регулярний вираз `re_kv`, щоб він коректно обробляв лапки з екранованими символами всередині.
    - Поточний: `(?P<key>[\w-]+)=\s*(?P<val>"[^"]*"|\S+)`
    - Новий: `(?P<key>[\w-]+)=\s*(?P<val>"(?:[^"\\]|\\.)*"|\S+)`

### 1.2. Реалізація `unescape_mikrotik(input: &str) -> String`
- [ ] Створити функцію для розкодування рядків:
    - `\\` -> `\`
    - `\"` -> `"`
    - `\n`, `\r`, `\t`, `\?`, `\$` -> відповідні символи.
    - `\HH` (де HH — дві шістнадцяткові цифри) -> байт з кодом HH.
    - Обробка послідовностей байтів як UTF-8.

### 1.3. Реалізація `escape_mikrotik(input: &str) -> String`
- [ ] Створити функцію для кодування рядків перед відправкою:
    - Екранування `\`, `"`, `$`, `?`.
    - Перетворення Unicode символів у послідовність `\HH` для кожного байта UTF-8.
    - Додавання лапок навколо рядка, якщо він містить пробіли або спецсимволи (або завжди для простоти).

## 2. Інтеграція в існуючий код

### 2.1. Оновлення `parse_all`
- [ ] Застосувати `unescape_mikrotik` до всіх значень, отриманих з регулярного виразу (якщо вони були в лапках або містять `\`).

### 2.2. Оновлення генерації запитів (`app.rs` та `mikrotik_data.rs`)
- [ ] Оновити `generate_find_query`, щоб всі значення (MAC, IP, коментар) проходили через `escape_mikrotik`.
- [ ] Оновити `save_lease` та `delete_lease`, щоб параметри команд були загорнуті в `escape_mikrotik`.

## 3. Тестування (TDD)

- [ ] Додати unit-тести для `unescape_mikrotik` з прикладами:
    - `\"Hello world\"`
    - `\D0\B0\D0\B1\D1\82` (кирилиця "абт")
    - `\?\$`
- [ ] Додати unit-тести для `escape_mikrotik`:
    - "Привіт" -> `\D0\9F\D1\80\D0\B8\D0\B2\D1\96\D1\82`
    - `Path with "quotes"` -> `Path with \"quotes\"`
- [ ] Перевірити роботу через `MockSSHClient`, імітуючи відповіді з екранованими символами.

## Питання до уточнення:
- Чи потрібно завжди загортати значення в лапки при відправці, чи лише коли вони містять спецсимволи/пробіли? (Зазвичай безпечніше завжди загортати в лапки).
  Відповідь: так, завжди загортати в лапки.
- Як MikroTik обробляє великі літери в шістнадцяткових кодах (`\AB` vs `\ab`)? (Зазвичай обидва варіанти працюють, але MikroTik часто видає великі).
  Відповідь: ROS7.x підтримує тільки великі шіснадцяткові літери. Маленькі літери викликають помилку.
