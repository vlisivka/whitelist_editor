# План впровадження валідації MAC та IP адрес

Цей план передбачає додавання механізмів перевірки коректності введених даних перед їх відправкою на роутер MikroTik.

## 1. Розширення бізнес-логіки (`mikrotik_data.rs`)
Згідно з принципами TDD, спочатку додамо тести, а потім реалізацію.

- [ ] Додати тести для нової функції `is_valid_mac(mac: &str) -> bool`.
- [ ] Реалізувати `is_valid_mac` (перевірка формату `XX:XX:XX:XX:XX:XX` через regex).
- [ ] Додати тести для `is_valid_ipv4(ip: &str) -> bool`.
- [ ] Реалізувати `is_valid_ipv4` (використовуючи стандартний парсинг `Ipv4Addr` та перевіряючи, чи адреса унікальна, чи попадає вона в потрібний діапазон адрес).

## 2. Посилення валідації форми (`app.rs`)
Необхідно оновити інтерфейс редагування, щоб користувач бачив помилки в реальному часі.

- [ ] Додати візуальну індикацію для полів введення в `show_edit_lease_window`:
    - Якщо MAC-адреса невалідна — підсвічувати поле червоним і виводити відповідне повідомлення.
    - Якщо IP-адреса невалідна, не унікальна або не входить в діапазон мережі — виводити відповідне повідомлення.
- [ ] Деактивувати (disable) кнопку "Зберегти", якщо хоча б одне з ключових полів містить помилку.

## 3. Автоматичне форматування (Optional/QoL)
- [ ] Додати автоматичне приведення MAC-адреси до верхнього регістру при введенні.
- [ ] Потрібно автоматично додавати двокрапки при введенні MAC-адреси.

## 4. Тестування (TDD)
- [ ] Написати GUI-тест у `app.rs`, який перевіряє, що кнопка "Зберегти" заблокована при введенні завідомо неправильної MAC-адреси.
- [ ] Написати GUI-тест для перевірки повідомлення про дублікат IP-адреси.
